# Project Rules

- Do not create or edit Unity prefabs unless the user explicitly asks for prefab work.
- When a request can be solved by adding a prefab or changing Inspector values, explain the configuration instead of changing code.
- Prefer extending an existing enemy, bullet, or player script over adding a new `.cs` file when the behavior fits the existing type.
- When code changes add Inspector fields, explain every new field and the values needed for the requested behavior.
